Static bevy_lint::EMPTY_QUERY

source · 
pub static EMPTY_QUERY: &Lint
Expand description

What it does: Detects empty Queries that will never return any Data. Also triggered when a Option is used, that will always return None.

Why is this bad? These Queries will never return any Data, because no Entity will fullfill the Requirements.

Known problems: None.

Example:

fn system(query: Query<&A, Without<A>>) {}

Instead do:

fn system(query: Query<&A>) {}
